Data Collection

This project does not involve external data collection.

Description:
All data used in the system is predefined within the application, including 
the list of grocery products.

Data Handling:
- User information, cart data, and orders are stored locally using browser Local Storage
- No external APIs or databases are used
- No user data is transmitted outside the browser

Purpose:
Local storage is used to simulate a database for demonstration and testing purposes.

Future Improvements:
A real-world version of this system could integrate external APIs or a backend database 
to handle live product data and user information securely.
